## üí° PRACTICAL EXAMPLES

### **Example 1: Explaining Your Process**

**CLIENT:** "How do you work?"

**WRONG (system-focused):**
"The company uses a 28-protocol workflow system with quality gates and evidence tracking."

**CORRECT (developer-focused):**
"I start by making sure I understand exactly what you need. Then I create a clear plan with milestones so you know what to expect. I keep you updated regularly and test everything before delivery. This way there are no surprises."

---

### **Example 2: Discussing Timeline**

**CLIENT:** "When can you start?"

**WRONG (passive):**
"The workflow allows for immediate initiation upon contract approval."

**CORRECT (active):**
"I can start as soon as we finalize the requirements. Based on what we've discussed, I'd need about 2 weeks for the core features. I'll show you working demos along the way."

---

### **Example 3: Addressing Quality**

**CLIENT:** "How do you ensure quality?"

**WRONG (process-focused):**
"Protocol 10, 11, and 12 include quality gates with automated validation scripts."

**CORRECT (outcome-focused):**
"I test everything thoroughly before showing it to you. I also do code reviews and make sure it works on different devices. You'll see working demos at each milestone - if something's not right, we catch it early."

---

### **Example 4: Handling Changes**

**CLIENT:** "What if we need changes?"

**WRONG (protocol-focused):**
"Protocol 08b handles change requests with impact analysis and approval workflows."

**CORRECT (human-focused):**
"Changes happen - I understand. When you need something different, I'll explain how it affects the timeline and budget, then we decide together. Everything stays documented so we're always aligned."
